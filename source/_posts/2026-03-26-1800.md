## 背景

之前博客一直使用 `hexo-filter-mermaid-diagrams` 插件来渲染 Mermaid 图表。然而，该插件已停止维护多年（最后更新于 2020 年），无法兼容新版 Hexo 和 Mermaid。

同时，该插件使用的是构建时渲染方案，需要安装 Puppeteer 等重型依赖，配置复杂，对于简单需求来说有些"杀鸡用牛刀"。

## 解决方案：hexo-markdown-mermaid

我开发了一个轻量级的 Hexo 插件 [hexo-markdown-mermaid](https://github.com/mangege/hexo-markdown-mermaid)，采用浏览器端渲染，零依赖，即装即用。

### 核心特点

1. **零依赖** - 无需 Puppeteer，仅通过 CDN 引入 Mermaid.js
2. **轻量级** - 核心代码仅约 20 行
3. **配置灵活** - 支持自定义 Mermaid 版本和配置参数

### 工作原理

插件通过 Hexo 的 `injector` 在页面底部注入 Mermaid.js 和初始化代码：

```javascript
hexo.extend.injector.register('body_end', `
<script src="https://cdn.jsdelivr.net/npm/mermaid@${version}/dist/mermaid.min.js"></script>
<script>
  mermaid.initialize(${JSON.stringify(config)});
</script>
`, 'default');
```

Mermaid 默认使用 `.mermaid` 选择器查找图表元素，因此只需确保代码块输出为 `<pre class="mermaid">` 即可自动渲染。

### 与语法高亮的兼容

如果使用 highlight.js，需要在 `_config.yml` 中排除 mermaid 语言：

```yaml
highlight:
  enable: true
  exclude_languages:
    - mermaid
```

原因：highlight.js 会预编译代码块为 HTML，生成 `<pre><code class="highlight mermaid">`，而 Mermaid 默认查找 `<pre class="mermaid">`。通过排除让 highlight.js 跳过 mermaid 块，保留原始格式供 Mermaid 渲染。

## 实现细节

```javascript
const mermaidVersion = hexo.config.mermaid?.version || '11';
const { version: _version, ...userConfig } = hexo.config.mermaid || {};
const mermaidConfig = Object.assign({
  startOnLoad: true
}, userConfig);

hexo.extend.injector.register('body_end', `
<script src="https://cdn.jsdelivr.net/npm/mermaid@${mermaidVersion}/dist/mermaid.min.js"></script>
<script>
  document.addEventListener('DOMContentLoaded', function() {
    mermaid.initialize(${JSON.stringify(mermaidConfig)});
  });
</script>
`, 'default');
```

关键点：
1. 从 Hexo 配置读取 `mermaid.version` 和其他参数
2. 使用展开运算符排除 `version` 字段，避免传入 Mermaid 初始化配置
3. 通过 `DOMContentLoaded` 确保 DOM 加载完成后初始化

## 高度定制化

如果插件无法满足需求，可以手动在主题中添加 Mermaid 支持。在 `scripts/` 目录下创建自定义脚本：

```javascript
hexo.extend.injector.register('body_end', `
<script src="https://cdn.jsdelivr.net/npm/mermaid@11/dist/mermaid.min.js"></script>
<script>
  mermaid.initialize({
    startOnLoad: true,
    theme: 'dark',
    securityLevel: 'loose',
    // 其他自定义配置...
  });
</script>
`, 'default');
```

这种方式可以：
- 完全自定义 Mermaid 配置
- 修改选择器（如 `querySelector: 'pre code.mermaid'`）
- 添加自定义样式或事件处理

## 安装使用

```bash
npm install hexo-markdown-mermaid
```

配置：

```yaml
mermaid:
  version: 11
  startOnLoad: true
  theme: default
```

详细文档见 [GitHub](https://github.com/mangege/hexo-markdown-mermaid)。
